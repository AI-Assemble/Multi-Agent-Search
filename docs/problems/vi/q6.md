# Q6 (4 điểm): Risk-aware Evaluation Function

Trong câu này, hãy thiết kế một hàm đánh giá state có ý thức về rủi ro tại `riskAwareEvaluationFunction` trong `src/core/agents/multiAgents.py`.

Khác với Q5 chỉ yêu cầu một evaluation function mạnh hơn nói chung, Q6 tập trung vào việc Pacman phải cân bằng giữa tiến độ ăn thức ăn và khả năng sống sót khi ghost đang ở gần. Hàm đánh giá nên:

* thưởng điểm cho score hiện tại, thức ăn gần, capsule gần, và ghost đang scared nếu Pacman có thể bắt kịp;
* phạt số lượng thức ăn/capsule còn lại để Pacman vẫn tiến về mục tiêu thắng;
* phạt mạnh các state mà ghost hoạt động có thể bắt Pacman ngay hoặc sau một bước di chuyển;
* phạt thêm khi Pacman bị kẹt trong vị trí ít đường thoát lúc ghost đang gần.

Phần sinh viên cần chỉnh/nộp là `riskAwareEvaluationFunction` tại `multiAgents.py`; starter code cố ý để TODO giống Q5 và không có công thức scoring mẫu. Alias `riskAware = riskAwareEvaluationFunction` được giữ để autograder truyền vào agent bằng `evalFn=riskAware`.

**Chấm điểm:** autograder chạy `ExpectimaxAgent` với `evalFn=riskAware` trên `smallClassic` với một ghost ngẫu nhiên trong 10 game. Điểm được cộng theo số game không timeout, số trận thắng, và điểm trung bình.

Chạy riêng Q6 bằng Taskfile:

```bash
task test:q6
```

Hoặc chạy trực tiếp autograder:

```bash
python -m autograder -q q6 --no-graphics
```
