# Q3 (5 điểm): Alpha-Beta Pruning

`Lưu ý về cách chạy:` Các lệnh bên dưới dùng `legacy-compatible mode` từ thư mục gốc. Nếu bạn làm việc theo cấu trúc mới, dùng `PYTHONPATH=src/core` cho Pacman và `PYTHONPATH="tests;src/core"` cho autograder.

Hãy tạo một agent mới dùng alpha-beta pruning để duyệt cây minimax hiệu quả hơn, trong `AlphaBetaAgent`. Một lần nữa, thuật toán của bạn cần tổng quát hơn pseudocode trên lớp, nên thử thách là mở rộng logic alpha-beta pruning cho nhiều minimizer agent.

Bạn sẽ thấy tốc độ cải thiện (có thể alpha-beta depth 3 chạy nhanh tương đương minimax depth 2). Lý tưởng là depth 3 trên `smallClassic` nên chạy chỉ vài giây mỗi nước đi hoặc nhanh hơn.
```bash
python -m src.core.controller.pacman -p AlphaBetaAgent -a depth=3 -l smallClassic
```
Hoặc dùng Taskfile launcher:

```bash
# Chạy AlphaBetaAgent với depth=3 bằng Taskfile
PACMAN_AGENT=AlphaBetaAgent PACMAN_LAYOUT=smallClassic PACMAN_EXTRA_ARGS='-a depth=3' task run:pacman
```
Giá trị minimax của `AlphaBetaAgent` phải giống với `MinimaxAgent`, dù action được chọn có thể khác do cách tie-break khác nhau.

**Chấm điểm:** Vì chúng tôi kiểm tra số lượng state mà mã của bạn mở rộng, bạn cần thực hiện alpha-beta pruning mà không đổi thứ tự các node con. Nói cách khác, successor state phải luôn được xử lý theo đúng thứ tự từ `GameState.getLegalActions`. Đồng thời, không được gọi `GameState.generateSuccessor` nhiều hơn mức cần thiết. Bạn không được prune khi bằng nhau để khớp đúng tập state mà autograder của chúng tôi kỳ vọng.

Để test và debug, chạy:
```bash
python -m autograder -q q3
```

Hoặc dùng Taskfile:

```bash
# Chạy autograder cho Q3 bằng Taskfile
task test:q3
```
