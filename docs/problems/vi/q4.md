# Q4 (5 điểm): Expectimax

`Lưu ý về cách chạy:` Các lệnh bên dưới dùng `legacy-compatible mode` từ thư mục gốc. Nếu bạn làm việc theo cấu trúc mới, dùng `PYTHONPATH=src/core` cho Pacman và `PYTHONPATH="tests;src/core"` cho autograder.

Minimax và alpha-beta rất tốt, nhưng cả hai đều giả định bạn đang đấu với đối thủ luôn ra quyết định tối ưu. Như bất kỳ ai từng thắng tic-tac-toe đều biết, điều này không phải lúc nào cũng đúng. Ở câu này, bạn sẽ cài `ExpectimaxAgent`, hữu ích để mô hình hóa hành vi xác suất của các agent có thể đưa ra lựa chọn không tối ưu.

Để tăng tốc quá trình tự phát triển, chúng tôi đã cung cấp một số test dựa trên cây tổng quát. Bạn có thể debug cài đặt của mình trên các cây game nhỏ bằng lệnh:
```bash
python autograder.py -q q4
```
Tương đương trong cấu trúc mới:
```bash
PYTHONPATH="tests;src/core" python -m autograder -q q4
```
`ExpectimaxAgent` sẽ không còn lấy min trên mọi hành động của ghost nữa, mà lấy kỳ vọng theo mô hình hành vi ghost của agent bạn. Để đơn giản hóa mã, giả sử bạn chỉ chạy với đối thủ chọn ngẫu nhiên đồng đều giữa các hành động trong `getLegalActions`.

Để xem `ExpectimaxAgent` hoạt động trong Pacman như thế nào, chạy:
```bash
python pacman.py -p ExpectimaxAgent -l minimaxClassic -a depth=3
```
Tương đương trong cấu trúc mới:
```bash
PYTHONPATH=src/core python -m controller.pacman -p ExpectimaxAgent -l minimaxClassic -a depth=3
```
Bạn sẽ thấy cách chơi liều hơn khi ở cự ly gần ghost.
