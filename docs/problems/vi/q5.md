# Q5 (6 điểm): Evaluation Function

`Lưu ý về cách chạy:` Các lệnh bên dưới dùng `legacy-compatible mode` từ thư mục gốc. Nếu bạn làm việc theo cấu trúc mới, dùng `PYTHONPATH="tests;src/core" python -m autograder ...` cho các lệnh autograder tương đương.

Hãy viết một hàm đánh giá tốt hơn cho Pacman trong hàm `betterEvaluationFunction` đã cho. Hàm đánh giá này cần đánh giá state, thay vì đánh giá action như reflex agent. Với search depth 2, hàm đánh giá của bạn nên vượt `smallClassic` với một ghost ngẫu nhiên hơn một nửa số lần, đồng thời vẫn chạy với tốc độ hợp lý (để đạt tối đa điểm, Pacman nên trung bình khoảng 1000 điểm khi thắng).

**Chấm điểm:** autograder sẽ chạy agent của bạn trên `smallClassic` 10 lần. Chúng tôi chấm điểm hàm đánh giá theo cách sau:
* Nếu bạn thắng ít nhất 1 lần mà không làm autograder timeout, bạn nhận 1 điểm. Agent nào không đáp ứng điều này sẽ nhận 0 điểm.
* +1 nếu thắng ít nhất 5 lần, +2 nếu thắng cả 10 lần.
* +1 nếu điểm trung bình ít nhất 500, +2 nếu điểm trung bình ít nhất 1000 (tính cả các ván thua).
* +1 nếu thời gian trung bình mỗi game dưới 30 giây trên máy autograder khi chạy với `--no-graphics`.

Bạn có thể thử agent trong các điều kiện trên bằng:
```bash
python autograder.py -q q5
```
