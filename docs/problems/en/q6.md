# Q6 (4 pts): Risk-aware Evaluation Function

For this question, design a risk-aware state evaluation function named `riskAwareEvaluationFunction` in `src/core/agents/multiAgents.py`.

Unlike Q5, which asks for a generally stronger evaluation function, Q6 focuses on balancing progress with survival when active ghosts are nearby. The function should:

* reward current score, nearby food, nearby capsules, and scared ghosts Pacman can realistically reach;
* penalize remaining food/capsules so Pacman still moves toward winning;
* strongly penalize states where an active ghost can catch Pacman immediately or after one ghost move;
* add extra risk when Pacman has few escape routes while an active ghost is nearby.

The student-facing code to edit and submit is `riskAwareEvaluationFunction` inside `multiAgents.py`; the starter is intentionally TODO-only like Q5 and does not include a sample scoring formula. The alias `riskAware = riskAwareEvaluationFunction` is kept so the autograder can pass it to the agent with `evalFn=riskAware`.

**Grading:** the autograder runs `ExpectimaxAgent` with `evalFn=riskAware` on `smallClassic` with one random ghost for 10 games. Points are awarded for non-timeout games, wins, and average score.
